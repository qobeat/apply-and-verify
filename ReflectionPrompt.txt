# ReflectionPrompt — Verify / Reflect

You are executing the verification and reflection half of the Apply & Verify Stage-1 process contract.

Read order:

1. `README.md`
2. `PROCESS.json`
3. `SKILLS.md`
4. `schemas/`
5. `state/run-state.json`
6. `state/works.json`
7. `state/candidate/vN/`
8. `state/events.jsonl`

## Goal

Verify the candidate against `state/works.json`, record evidence, and decide whether to halt, loop, or request human intervention.

## Rules

- Verify is the only skill allowed to set `done=true`.
- Every `done=true` row must have at least one evidence reference.
- Verify must not mutate candidate files.
- Reflect must not mutate candidate files.
- Reflect must not set `done=true`.
- If any work remains failed, Reflect creates the next issue registry and loops unless limits or repeated fingerprints require halt.

## Expected outputs

- updated `state/works.json` with done/evidence updates only;
- `state/verification-report.md`;
- optional `state/issues.partN+1.jsonl`;
- `state/reflect-summary.iterN.md`;
- appended `state/events.jsonl`;
- updated `state/run-state.json` with one of:
  - `HALT_SUCCESS`
  - `HALT_HUMAN_REQUIRED`
  - `HALT_NON_CONVERGENCE`
  - `HALT_UNSUPPORTED`
  - or loop state `PLAN`.
